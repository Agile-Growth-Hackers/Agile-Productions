# Security Policy

## Reporting a Vulnerability

We take the security of our project seriously. If you discover a security vulnerability, please follow these steps:

### How to Report

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via one of the following methods:

1. Contact us directly at: https://agilegrowthhackers.com
2. If possible, encrypt your message using our PGP key (if available)

### What to Include

When reporting a vulnerability, please include:

- Type of vulnerability
- Full paths of source file(s) related to the vulnerability
- Location of the affected source code (tag/branch/commit or direct URL)
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the vulnerability, including how an attacker might exploit it

### Response Timeline

- We will acknowledge receipt of your vulnerability report within 48 hours
- We will provide a detailed response within 7 days, including next steps
- We will keep you informed of the progress towards resolving the vulnerability
- We may ask for additional information or guidance

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| < Latest| :x:                |

## Security Updates

Security updates will be released as soon as possible after a vulnerability is confirmed and a fix is available.

## Disclosure Policy

- We ask that you do not publicly disclose the vulnerability until we have had a chance to address it
- We will credit you for the discovery (unless you prefer to remain anonymous)
- We aim to resolve critical vulnerabilities within 30 days

Thank you for helping keep our project and users safe!
